Ajax

原生 走服务端 直连 带附件
  • 用法
  • $util.form(options)
  • options的参数请参考原生ajax
  • 重要说明
  • options比其他ajax多一个属性：fileElementId
    此属性指明存储文件地址的控件id，其值可以是一个字符串（仅一个文件需要提交），也可以是一个数组（多个文件需要提交）
  • 说明
  • 1、url地址必须是完整路径http|https地址
  • 2、此请求直接与url对应的服务器连接，不经过ExMobi服务端，所以不会维持于第三方session
  • 3、具体功能请看ExMobi的DirectFormSubmit对象
  • 4、如果不需要提交附件，仅有参数建议使用$util.ajax
  • 用法
  • $util.ajax(options)
  • options的参数请参考原生ajax
  • 说明
  • 1、url地址必须是完整路径http|https地址
  • 2、此请求直接与url对应的服务器连接，不经过ExMobi服务端，所以不会维持于第三方session
  • 3、具体功能请看ExMobi的DirectAjax对象
  • 用法
  • $util.server(options)
  • options的参数请参考原生ajax
  • 说明
  • 1、url地址必须是完整路径http|https地址
  • 2、此请求会经过ExMobi服务端，所以会维持于第三方session
  • 3、具体功能请看ExMobi的Ajax对象描述
  • Zepto|jQuery
  • 采用的是Zepto或jQuery
  • 用法：$.ajax(options)
  • Agile
  • Agile也封装建议Ajax调用
  • 用法：A.ajax(options)
  • options参数
  • options为一个json对象，包含：url、type、data、dataType、jsonp、success、error
  • url
  • 要请求的url地址
  • type
  • 请求的方法，get|post
  • data
  • 提交的参数，可以是键值对，也可以是xml或者json
  • dataType
  • 相应数据类型，text|json|jsonp
  • jsonp
  • 设定jsonp提交的函数获取参数名
  • success
  • 请求成功回调函数
  • error
  • 请求失败回调函数
